# TalkFlow 실서비스 출시 기준서

이 문서는 TalkFlow를 실제 사용자에게 공개하기 전에 반드시 확인해야 할 출시 기준을 정의합니다.

## 1. 출시 판단 기준

TalkFlow는 아래 세 가지 기준을 모두 충족해야 실서비스로 공개할 수 있습니다.

1. 핵심 학습 플로우가 안정적으로 동작한다.
2. 사용자 데이터와 AI 응답이 안전하게 관리된다.
3. 오류가 발생해도 사용자가 막히지 않고 회복할 수 있다.

## 2. 출시 단계 구분

| 단계 | 공개 범위 | 목표 |
|---|---|---|
| 내부 테스트 | 팀/개발자 | 기능 정상 동작 확인 |
| 클로즈드 베타 | 제한된 사용자 | 실제 사용성/오류 확인 |
| 오픈 베타 | 링크 접근 사용자 | 운영 안정성 확인 |
| 정식 출시 | 일반 사용자 | 지속 운영 가능 상태 |

## 3. 출시 전 필수 조건

### 반드시 완료

1. 기본 시나리오 5개 제공
2. 레벨 선택 반영
3. 말하기 속도 설정 반영
4. 음성 또는 키보드 입력 가능
5. 학습자 발화 평가 가능
6. AI 다음 질문 생성 가능
7. 학습 리포트 생성 가능
8. 0턴 / 1~4턴 / 5턴 이상 / 10턴 종료 처리
9. 오류 문구와 재시도 흐름 제공
10. 사용자 데이터 저장/삭제 기준 마련

### 출시 보류 조건

아래 중 하나라도 해당하면 출시를 보류합니다.

1. AI가 한 턴에 여러 질문을 자주 생성함
2. 의료 상황에서 진단/처방처럼 보이는 답변을 생성함
3. 마이크 실패 시 키보드 대체가 불가능함
4. 리포트 수치가 실제 대화와 다르게 표시됨
5. 사용자 발화나 리포트가 의도치 않게 사라짐
6. API 키가 프론트엔드에 노출됨
7. 개인정보 처리방침이 없음

실서비스 전 아래 문서 또는 안내가 필요합니다.

- 음성 데이터 처리 방식
- 학습자 발화 텍스트 저장 여부

- 데이터 보관 기간

### 이용약관

포함해야 할 내용:

- 서비스 목적

- 서비스 중단 가능성

### AI 고지

앱 내 또는 약관에 아래 내용을 명확히 안내합니다.

```text
TalkFlow의 AI 피드백은 영어 학습을 돕기 위한 참고용 안내입니다. 모든 표현이 유일한 정답은 아니며, 상황에 따라 다르게 쓰일 수 있습니다.
```

### 음성 데이터 고지

```text
음성 입력은 영어 말하기 연습을 위해 텍스트로 변환됩니다. MVP에서는 원본 음성 파일을 기본 저장하지 않습니다.
```

### 의료/전문 조언 고지

```text
병원 등 의료 상황의 대화는 영어 연습용입니다. 실제 진단이나 치료 판단은 의료 전문가와 상담해 주세요.
```

## 7. 보안 기준

### 필수

1. AI API 키는 서버에만 저장한다.
2. 프론트엔드에 비밀 키를 노출하지 않는다.
3. HTTPS로 배포한다.
4. 사용자 입력값을 검증한다.
5. API 요청 횟수를 제한한다.
6. 오류 로그에 민감 정보를 남기지 않는다.
7. 관리자 기능은 인증 뒤 접근하게 한다.

### 권장

- Rate limiting
- Abuse detection
- 요청 크기 제한
- CORS 제한
- 서버 환경변수 분리
- 개발/스테이징/프로덕션 환경 분리

## 8. AI 안전 기준

### 공통

- 학습자의 레벨을 반영한다.
- 한 턴에 질문은 하나만 한다.
- 1턴은 AI 발화 1회와 학습자 발화 1회를 합친 한 쌍으로 계산한다.
- 주제 이탈 시 상황으로 되돌린다.
- 부정확한 문장도 비난하지 않는다.
- 한국어 피드백은 친절한 EFL 교사 말투로 작성한다.

### 금지 또는 제한

AI는 아래를 하지 않습니다.

- 의료 진단
- 약 처방
- 법률 판단
- 위험 행동 조언
- 혐오/차별 표현 생성
- 성적/폭력적 역할극 생성
- 개인정보 요구
- 미성년자에게 부적절한 콘텐츠 생성

### 의료 시나리오 안전선

허용:

- 증상을 영어로 설명하는 연습
- 병원 접수/진료 상황에서 쓸 수 있는 일반 표현

제한:

- 병명 추정
- 약 복용 지시
- 응급 여부 판단

## 9. 콘텐츠 운영 기준

### 기본 시나리오 검수

각 시나리오는 아래 기준을 통과해야 합니다.

1. 역할이 명확한가?
2. 미션이 실제 대화에서 수행 가능한가?
3. 표현과 단어가 미션과 직접 연결되는가?
4. 5턴 안에 미션 완료가 가능한가?
5. 6~10턴 확장이 자연스러운가?
6. 레벨에 맞는 난이도인가?
7. 민감하거나 위험한 내용이 없는가?

### AI 생성 시나리오 검수

나만의 시나리오 생성 결과는 아래 항목을 검사합니다.

- 금지 주제 포함 여부
- 역할 누락 여부
- 미션 수
- 질문 수
- 레벨 난이도
- JSON 구조 유효성

## 12. QA 출시 기준

`qa-test-plan.md` 기준으로 아래 조건을 충족해야 합니다.

### 최소 통과 기준

- 필수 테스트 케이스 90% 이상 통과
- 대화 종료 관련 테스트 100% 통과
- 리포트 계산 테스트 100% 통과
- 마이크 실패 대체 흐름 100% 통과
- 모바일 주요 화면 텍스트 잘림 없음

### 출시 보류 QA 항목

- 대화가 중간에 멈추고 복구할 수 없음
- 리포트가 실제 대화와 다름
- 버튼이 모바일에서 눌리지 않음
- 키보드 입력창이 대화 전체를 가림
- AI 질문이 여러 개씩 나와 학습자가 혼란스러움

AI와 음성 기능은 비용이 발생하므로 제한 정책이 필요합니다.

- 비정상 반복 요청 차단

- 실패한 AI 호출 수
- 평균 대화 턴 수
