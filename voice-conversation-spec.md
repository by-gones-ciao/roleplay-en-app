# TalkFlow 음성/대화 기능 기준서

이 문서는 TalkFlow 웹앱에서 실제 음성 대화 기능을 구현하기 위한 기능 기준입니다. 마이크 입력, 음성 인식, 키보드 입력, 다시 녹음, AI 음성 재생, 번역, 대화 종료, 리포트 이동까지 포함합니다.

## 1. 핵심 사용자 흐름

### 턴 정의

- 1턴은 AI 발화 1회와 학습자 발화 1회를 합친 한 쌍을 의미한다.
- 예: AI가 질문하고 학습자가 답하면 1턴이 완료된다.
- `turnCount`는 완료된 AI-학습자 대화 쌍의 개수를 의미한다.
- AI 발화만 표시되고 학습자가 아직 답하지 않은 상태는 완료된 1턴으로 계산하지 않는다.

### 기본 흐름

1. 사용자가 `대화 시작하기`를 누른다.
2. AI가 첫 질문을 보여준다.
3. 사용자가 마이크 버튼을 눌러 말한다.
4. 앱이 음성을 텍스트로 변환한다.
5. 앱이 학습자 발화를 평가한다.
6. 학습자 말풍선과 피드백 아이콘을 표시한다.
7. AI가 다음 질문을 생성한다.
8. AI 말풍선, 번역 버튼, 음성 버튼을 표시한다.
9. 레벨별 최소/최대 학습자 답변 수 기준에 따라 대화를 이어간다.
10. 종료 시 학습 리포트를 제공한다.

### 대체 흐름

- 마이크를 사용할 수 없으면 키보드 입력으로 대체한다.
- 음성 인식에 실패하면 다시 말하기 또는 키보드 입력을 제공한다.
- 사용자가 대화 도중 X를 누르면 진행 정도에 맞게 종료 확인을 제공한다.

## 2. 마이크 권한 흐름

### 첫 마이크 사용 시

브라우저 또는 앱 권한 요청을 띄운다.

상태:

| 상태 | 앱 동작 |
|---|---|
| 권한 허용 | 녹음 시작 |
| 권한 거부 | 안내 문구 + 키보드 입력 제공 |
| 권한 대기 | 로딩 상태 표시 |
| 권한 알 수 없음 | 다시 시도 버튼 제공 |

### 권한 거부 문구

```text
마이크 권한이 꺼져 있어요. 설정에서 권한을 허용하거나 키보드로 입력해 주세요.
```

버튼:

- 다시 시도
- 키보드로 입력

## 3. 녹음 상태

### 상태값

```json
{
  "recordingState": "idle | requesting_permission | recording | processing | failed"
}
```

### 상태별 UI

| 상태 | UI |
|---|---|
| idle | 마이크 버튼 표시 |
| requesting_permission | 권한 확인 중 표시 |
| recording | 녹음 중 애니메이션, 정지 버튼 |
| processing | 음성을 확인 중이에요 |
| failed | 다시 말하기 / 키보드 입력 |

### 녹음 중 문구

```text
듣고 있어요.
```

### 처리 중 문구

```text
말한 내용을 확인하고 있어요.
```

## 4. 음성 인식 기준

### 성공 조건

- 최소 1개 이상의 영어 단어가 인식됨
- 공백만 있지 않음
- 너무 짧은 잡음이 아님

### 실패 조건

| 실패 유형 | 기준 | 문구 |
|---|---|---|
| no_speech | 아무 말도 감지되지 않음 | 말소리가 잘 들리지 않았어요. 다시 말해 주세요. |
| too_short | 너무 짧음 | 조금만 더 길게 말해 주세요. |
| unclear | 인식 결과가 불명확함 | 정확히 듣지 못했어요. 다시 말하거나 키보드로 입력해 주세요. |
| language_mismatch | 영어가 거의 없음 | 영어로 한 문장 말해 주세요. |
| network_error | 서버/네트워크 오류 | 연결이 불안정해요. 잠시 후 다시 시도해 주세요. |

### 실패 시 버튼

- 다시 말하기
- 키보드로 입력

## 5. 키보드 입력 흐름

### 진입 방법

- 하단 키보드 버튼을 누른다.
- 마이크 실패 화면에서 `키보드로 입력`을 누른다.
- 마이크 권한 거부 시 `키보드로 입력`을 누른다.

### UI 기준

- 하단에 입력창이 올라온다.
- 전체 대화창은 가려지지 않아야 한다.
- 입력창, 보내기 버튼, 닫기 버튼을 제공한다.
- 빈 입력은 전송하지 않는다.

### 빈 입력 문구

```text
영어 문장을 입력해 주세요.
```

## 6. 학습자 발화 저장 기준

학습자 발화는 음성 입력이든 키보드 입력이든 동일한 구조로 저장한다.

```json
{
  "turnIndex": 3,
  "inputMode": "voice | keyboard",
  "rawTranscript": "At the same level my eyebrows.",
  "finalText": "At the same level my eyebrows.",
  "createdAt": "2026-07-10T09:00:00+09:00"
}
```

### 저장 기준

- rawTranscript: 음성 인식 원문
- finalText: 사용자가 확정한 문장
- keyboard 입력은 rawTranscript와 finalText가 같아도 된다.

## 7. 학습자 발화 평가 흐름

1. 학습자 발화를 수집한다.
2. `학습자 발화 평가 프롬프트`에 보낸다.
3. 결과를 `perfect`, `better`, `correction` 중 하나로 받는다.
4. 말풍선 앞에 아이콘을 표시한다.
5. perfect는 설명 팝업을 제공하지 않는다.
6. better/correction은 아이콘을 누르면 피드백 박스를 연다.

### 피드백 UI

#### perfect

- 아이콘: 초록 체크
- 설명 팝업 없음

#### better

- 아이콘: 손바닥
- 피드백 박스 순서:
  1. 한글 설명
  2. 구분선
  3. 더 좋은 표현
  4. 영어 문장

#### correction

- 아이콘: 빨간 느낌표
- 피드백 박스 순서:
  1. 한글 설명
  2. 구분선
  3. 교정 문장
  4. 영어 문장

## 8. 다시 녹음 기능

### 동작 기준

학습자 말풍선 옆의 새로고침 버튼은 해당 턴부터 다시 대답할 수 있게 한다.

예:

- 특정 학습자 답변을 다시 녹음하면 해당 답변 이후 대화는 삭제된다.
- 해당 턴의 AI 질문은 유지된다.
- 사용자는 같은 AI 질문에 다시 답한다.
- 이후 AI 응답과 리포트는 새 답변 기준으로 다시 계산한다.

### 삭제 범위

```text
선택한 학습자 턴 이후의 학습자 발화, AI 발화, 피드백, 미션 진행 상태를 다시 계산한다.
```

### 상태 초기화

- turnCount 재설정
- completedMissionIds 재계산
- feedbackResults 재계산
- continueConversation false
- finalizingConversation false

## 9. AI 응답 생성 흐름

1. 학습자 발화 평가가 끝난다.
2. 미션 완료 여부를 업데이트한다.
3. 현재 턴수와 대화 규칙을 확인한다.
4. `대화 중 AI 응답 생성 프롬프트`로 다음 AI 발화를 생성한다.
5. AI 발화는 한 턴에 질문 하나만 포함한다.
6. AI 말풍선과 도구 버튼을 표시한다.

### AI 응답 상태

| 상태 | 문구 |
|---|---|
| generating | 다음 질문을 준비하고 있어요. |
| failed | AI 응답을 가져오지 못했어요. |

### 실패 시 버튼

- 다시 시도
- 리포트 보기

## 10. AI 음성 재생

### 동작

- AI 말풍선 아래 음성 버튼을 누르면 해당 문장을 재생한다.
- 사용자가 설정한 말하기 속도를 반영한다.
- 재생 중 다시 누르면 정지한다.
- 다른 AI 음성을 누르면 기존 재생을 멈추고 새 음성을 재생한다.

### 속도 매핑

| 앱 표시 | 내부 값 | 재생 속도 |
|---|---:|---:|
| Slow(x0.8) | slow | 0.8 |
| A Little Slow(x0.9) | little_slow | 0.9 |
| Normal(x1.0) | normal | 1.0 |
| A Little faster(x1.2) | little_faster | 1.2 |

### 음성 실패 문구

```text
음성을 재생하지 못했어요. 잠시 후 다시 시도해 주세요.
```

## 11. 번역 버튼

### 동작

- AI 말풍선 아래 번역 버튼을 누르면 한국어 번역을 펼친다.
- 다시 누르면 접는다.
- 학습자 발화에는 번역을 제공하지 않는다.
- 학습 리포트의 전체 대화 보기에서도 AI 발화 번역을 제공한다.

### 기준

- 번역은 AI 발화 의미를 자연스럽게 보여준다.
- 직역보다 학습자가 이해하기 쉬운 번역을 우선한다.

## 12. 대화 진행 표시

### 기본 표시

- 최소 답변 수 전: `답변 진행 4/5`처럼 현재 답변 수와 최소 기준을 표시한다.
- 최소 답변 수 이후: `더 연습 중 6/10` 또는 `더 연습 중 12/20`처럼 현재 답변 수와 최대 기준을 표시한다.
- 마무리 예고 이후: `마무리 단계 9/10` 또는 `마무리 단계 18/20`처럼 표시한다.

### 진행바

- 최소 답변 수 전에는 레벨별 최소 기준으로 표시한다.
- 더 연습 중에는 레벨별 최대 기준으로 표시한다.
- 진행바는 한 줄로 표시한다.
- 텍스트는 최소화한다.

## 13. 미션 완료 표시

### 기준

- 미션은 의미 기반으로 완료 판정한다.
- 키워드만으로 판단하지 않는다.
- 미션 완료 시 번호가 체크 표시로 바뀐다.
- 기존 진행률 바는 사용하지 않는다.

### 예

```json
{
  "completedMissionIds": ["mission_1", "mission_2"]
}
```

## 14. 대화 종료 조건

### 0턴 종료

사용자가 아무 말도 하지 않고 X를 누른 경우:

문구:

```text
아직 대화를 시작하지 않았어요. 나가시겠어요?
```

버튼:

- 계속 대화하기
- 나가기

동작:

- 리포트 제공 없음
- 직전 진입 화면으로 이동

### 1회 이상 답변 후 종료

사용자가 1회 이상 답변했지만 레벨별 최소 답변 수 전에 종료한 경우:

문구:

```text
대화를 여기까지 할까요?
```

버튼:

- 계속하기
- 리포트 보기

동작:

- 짧은 리포트 제공
- 미션 완료율은 실제 완료 기준으로 계산

### 미션 완료 + 레벨별 최소 답변 수 이상

문구:

```text
미션을 모두 완료했어요!
마무리할까요 아니면 대화를 좀 더 할까요?
```

버튼:

- 마무리하고 리포트 보기
- 조금 더 대화하기

### 마무리 예고 시점

AI 마무리 예고:

```text
좋아요. 이제 마지막으로 하나만 더 확인해 볼게요.
```

### 레벨별 최대 답변 수

AI 종료 문구:

```text
오늘은 여기까지 대화해요. 학습 리포트를 확인해 보세요.
```

동작:

- 자동으로 학습 리포트로 이동

## 15. 학습 리포트 이동 기준

리포트는 다음 경우에 제공한다.

| 조건 | 리포트 제공 |
|---|---|
| 0턴 종료 | 제공 안 함 |
| 1회 이상 답변 후 조기 종료 | 짧은 리포트 |
| 레벨별 최소 답변 수 이상 종료 | 일반 리포트 |
| 미션 완료 후 종료 | 완료 리포트 |
| 레벨별 최대 답변 수 자동 종료 | 완료/진행 상태에 따른 리포트 |

## 16. 네트워크 오류

### 발생 가능 지점

- 음성 인식
- 학습자 발화 평가
- AI 응답 생성
- 음성 합성
- 리포트 생성

### 공통 문구

```text
연결이 불안정해요. 잠시 후 다시 시도해 주세요.
```

### 버튼

- 다시 시도
- 키보드로 입력
- 리포트 보기

단, `리포트 보기`는 완료 턴이 1턴 이상 있을 때만 제공한다.

## 17. 로딩 상태

### 로딩 문구

| 상황 | 문구 |
|---|---|
| 마이크 권한 확인 | 마이크 권한을 확인하고 있어요. |
| 녹음 처리 | 말한 내용을 확인하고 있어요. |
| 평가 생성 | 표현을 확인하고 있어요. |
| AI 응답 생성 | 다음 질문을 준비하고 있어요. |
| 리포트 생성 | 학습 리포트를 만들고 있어요. |

### 기준

- 1초 이상 걸리면 로딩 문구를 보여준다.
- 8초 이상 걸리면 재시도 버튼을 제공한다.

## 18. 접근성 기준

- 마이크, 키보드, 힌트, 번역, 음성 버튼에는 aria-label을 제공한다.
- 색상만으로 상태를 구분하지 않는다.
- 피드백 아이콘은 모양과 색을 함께 사용한다.
- 키보드 입력은 Enter 전송을 지원한다.
- 모바일 화면에서 입력창이 대화 내용을 완전히 가리지 않도록 한다.

## 19. 개인정보/저장 기준

### 저장 가능 데이터

- 학습자 발화 텍스트
- 발화 평가 결과
- 미션 완료 여부
- 대화 시간
- 리포트 결과

### 주의 데이터

- 원본 음성 파일
- 마이크 권한 상태
- 의료/민감 상황의 구체적 개인정보

### 기준

- MVP에서는 원본 음성 파일을 저장하지 않는 것을 기본으로 한다.
- 필요한 경우 사용자 동의 후 저장한다.
- 의료 상황에서는 실제 진단 정보를 수집하지 않도록 주의한다.

## 20. 개발 테스트 체크리스트

1. 첫 마이크 권한 요청이 정상 동작하는가?
2. 권한 거부 시 키보드 입력으로 대체되는가?
3. 음성 인식 실패 시 다시 말하기가 가능한가?
4. 키보드 입력이 전체 대화창을 가리지 않는가?
5. 빈 입력을 전송하지 않는가?
6. 학습자 발화가 perfect/better/correction으로 표시되는가?
