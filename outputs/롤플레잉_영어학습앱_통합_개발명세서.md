# TalkFlow 롤플레잉 영어학습앱 통합 개발명세서

작성일: 2026-07-20  
대상: 웹앱 MVP 개발자, AI 기능 개발자, 콘텐츠 제작자

## 1. 제품 개요

TalkFlow는 성인 한국인 영어 학습자가 AI와 역할극을 하며 실제로 말해 보는 영어 스피킹 학습 앱입니다. 핵심 경험은 다음 순서로 구성합니다.

1. 온보딩에서 영어 말하기 수준과 말하기 속도를 선택한다.
2. 홈에서 AI가 추천하는 오늘의 대화 또는 시나리오 리스트를 선택한다.
3. 오늘의 표현과 오늘의 단어를 학습한다.
4. 대화 직전 리마인드 화면에서 미션과 답변 수 기준을 확인한다.
5. 실제 대화하기 화면에서 AI와 역할극을 진행한다.
6. 대화 종료 후 학습 리포트에서 발화, 교정, 더 좋은 표현, 전체 대화를 확인한다.

## 2. 현재 구현 파일

| 파일 | 역할 |
|---|---|
| `index.html` | 앱 진입 HTML |
| `app.js` | 화면 상태, 학습 플로우, 샘플 데이터, 대화 규칙 |

현재 앱은 정적 프론트엔드 프로토타입입니다. 실제 AI, STT, TTS, 회원, DB, 결제, 알림 서버 기능은 추후 개발 범위입니다.

## 3. 학습 플로우

### AI 추천 오늘의 대화

1. 홈
2. 오늘의 표현
3. 오늘의 단어
4. 오늘의 대화 리마인드
5. 대화하기
6. 학습 리포트

### 시나리오 상세 진입

1. 홈
2. 시나리오 상세
3. 학습 후 대화하기 또는 바로 대화하기
4. 오늘의 표현/단어 또는 바로 대화 리마인드
5. 대화하기
6. 학습 리포트

대화 시작 전 화면에서 나가기를 선택하면 직전 진입 화면으로 돌아갑니다. 대화가 1회 이상 진행된 뒤 나가기를 선택하면 리포트 보기를 제공합니다.

## 4. 턴과 답변 수 정의

이 앱에서 1턴은 AI 발화 1회와 학습자 답변 1회를 합친 한 쌍입니다.

```text
AI: 질문 또는 반응
학습자: 답변
= 1턴 완료
```

따라서 `turnCount`는 단일 말풍선 수가 아니라 완료된 AI-학습자 대화 쌍의 개수이며, 학습자가 실제로 확정한 답변 수와 같습니다.

## 5. 레벨별 최소/최대 학습자 답변 수

| 레벨 | 최소 학습자 답변 수 | 최대 학습자 답변 수 | 마무리 예고 | 자동 종료 |
|---|---:|---:|---:|---:|
| Starter(입문) | 5회 | 10회 | 9회 이후 | 10회 |
| Beginner(초급) | 5회 | 20회 | 18회 이후 | 20회 |
| Pre-Intermediate(초중급) | 6회 | 20회 | 18회 이후 | 20회 |
| Intermediate(중급) | 7회 | 20회 | 18회 이후 | 20회 |
| Advanced(고급) | 8회 | 20회 | 18회 이후 | 20회 |

### 종료 규칙

- 미션 완료 전에는 최소 답변 수 전에 자동 종료하지 않습니다.
- 미션 완료 + 최소 답변 수 이상이면 종료 선택지를 보여줍니다.
- 사용자가 계속하기를 선택하면 레벨별 최대 답변 수까지 확장합니다.
- 마무리 예고 시점부터 AI는 자연스럽게 대화를 정리합니다.
- 최대 답변 수에 도달하면 AI가 마지막 안내 후 학습 리포트로 이동합니다.
- 0회 답변 상태에서 X를 누르면 리포트를 제공하지 않습니다.
- 1회 이상 답변 상태에서 X를 누르면 리포트 보기를 제공합니다.

## 6. 대화 화면 UX 기준

### 대화 직전 화면

- 제목은 `오늘의 대화`로 중앙 정렬합니다.
- 보조 문구: `틀려도 괜찮아요. 편하게 말해 보세요.`
- 설명 문구: `오늘 배운 표현과 단어를 사용해 미션을 완료해 보세요.`
- 오늘의 미션은 가벼운 리마인드 카드로 표시합니다.
- 레벨별 최소/최대 답변 수 안내를 버튼 위에 표시합니다.

### 실제 대화하기 화면

- 제목은 `대화하기`로 좌측 정렬합니다.
- 상단 오른쪽에 X를 둡니다.
- X 클릭 시 확인창을 보여줍니다.
  - `대화를 여기까지 할까요?`
  - `계속하기`
  - `리포트 보기`
- 미션 카드 안에는 `상황: ...`을 표시합니다.
- 미션 완료 시 진행률바 대신 미션 번호가 체크 표시로 바뀝니다.
- 턴 진행은 텍스트를 최소화하고 숫자 중심으로 보여줍니다.
  - 예: `8/10`, `12/20`

## 7. AI 대화 생성 기준

AI는 학습자가 미션을 의식하지 않아도 자연스럽게 미션을 완료하도록 대화를 이끌어야 합니다.

공통 원칙:

- AI는 한 턴에 반드시 하나의 질문만 합니다.
- 문장이 여러 개일 수는 있지만, 학습자에게 답해야 하는 질문은 하나만 둡니다.
- 특히 입문/초급에서는 yes/no 질문을 기본적으로 피합니다.
- 학습자가 주제를 벗어나면 짧게 인정한 뒤 자연스럽게 원래 상황으로 돌아옵니다.
- 레벨이 낮을수록 AI가 더 많이 리드합니다.
- 의료 상황에서는 진단, 처방, 전문적 의학 조언을 하지 않습니다.
- 대화의 시작과 끝맺음은 기본적으로 AI가 담당합니다.

## 8. 피드백 기준

학습자 발화는 세 가지 상태 중 하나로 평가합니다.

| 상태 | 의미 | UI |
|---|---|---|
| perfect | 의미와 표현이 자연스럽고 충분히 정확함 | 초록 체크 아이콘, 별도 설명 없음 |
| better | 의미 전달은 되지만 더 자연스러운 표현이 있음 | 손바닥 아이콘, 더 좋은 표현 박스 |
| correction | 문법, 어휘, 시제, 관사, 수 일치, 단수/복수, 전치사, 문장 구조 등 교정 필요 | 빨간 느낌표 아이콘, 교정 문장 박스 |

공통 피드백 원칙:

- 입문/초급도 오류 교정은 정확해야 합니다.
- 단, 설명은 레벨에 맞게 쉽고 짧게 제공합니다.
- 한국어 피드백은 친절한 EFL 교사 말투로 작성하고 `~요.`로 끝냅니다.
- better는 더 원어민다운 표현, 더 좋은 문장 구조, 자연스러운 능동/수동, 어휘 선택까지 포함합니다.
- correction은 문법, 어휘, 문장 구조, 시제, 관사, 수 일치, 단수/복수 등을 정확히 다룹니다.

## 9. 학습 리포트 기준

학습 리포트는 실제 대화 진행 정도에 따라 내용과 톤을 달리합니다.

| 상태 | 리포트 제공 | 톤 |
|---|---|---|
| 0회 답변 | 제공 안 함 | 없음 |
| 최소 답변 수 미만 | 제공 | 짧고 가벼운 리포트 |
| 최소 답변 수 이상 | 제공 | 일반 리포트 |
| 미션 모두 완료 | 제공 | 완료 리포트 |
| 최대 답변 수 자동 종료 | 제공 | 진행 상태에 따른 리포트 |

리포트 구성:

- 말한 단어
- 대화 시간
- 미션 평가
- 교정 문장 보기
- 더 좋은 표현 보기
- 전체 대화 보기
- AI 피드백

교정 문장 보기와 더 좋은 표현 보기는 실제 대화창 형식으로 보여줍니다. 전체 대화 보기에서는 AI 발화의 번역과 음성 버튼도 실제 대화창과 동일하게 제공합니다.

## 11. 입문 콘텐츠 제작 기준

입문은 영어로 입을 떼고 자신감을 얻는 단계입니다.

- 단어 조합 또는 아주 쉬운 문장으로 오류 없는 의사소통을 목표로 합니다.
- 실제 회화에서 자연스러운 서비스 chunk는 허용합니다.
  - 예: `One coffee, please.`
- 어색한 생략 표현은 사용하지 않습니다.
  - 예: `Bus to __?`는 제외
- 오늘의 표현은 미션을 수행할 수 있어야 합니다.
- 오늘의 단어는 학습자가 실제로 말할 단어를 우선합니다.
- 한국인 학습자가 이미 널리 아는 단어는 오늘의 단어로 굳이 넣지 않습니다.
  - 예: hello, thank you, coffee, cake, one-four, popcorn
- 단, cat, dog처럼 쉬워도 실제 한국어 생활에서는 영어로 덜 쓰는 단어는 학습 대상이 될 수 있습니다.
- 단어 예문은 짧고 직관적이어야 하며, 한 롤플레잉 안에서 예문 패턴 반복을 피합니다.

## 13. API 구현 초안

필요 API:

| API | 목적 |
|---|---|
| `POST /api/scenarios/generate` | 나만의 시나리오 생성 |
| `POST /api/conversation/evaluate` | 학습자 발화 평가 |

## 15. QA 필수 체크

- 레벨별 최소/최대 답변 수가 화면과 AI 동작에 동일하게 반영되는가?
- 1턴이 AI 1회 + 학습자 1회 기준으로 계산되는가?
- 입문은 최소 5회, 최대 10회로 제한되는가?
- 초급은 최소 5회, 최대 20회로 동작하는가?
- 초중급은 최소 6회, 최대 20회로 동작하는가?
- 중급은 최소 7회, 최대 20회로 동작하는가?
- 고급은 최소 8회, 최대 20회로 동작하는가?
- 미션 완료 전 최소 답변 수 이전에 자동 종료되지 않는가?
- 미션 완료 후 최소 답변 수 이상에서 종료 선택지가 보이는가?
- 최대 답변 수에서 자동으로 리포트로 이동하는가?
- 대화 중 X 종료 시 0회 답변과 1회 이상 답변 흐름이 다르게 처리되는가?
- 학습 리포트가 실제 대화 내용만 반영하는가?
- AI가 한 턴에 질문을 하나만 하는가?
- 입문/초급에서 yes/no 질문으로 대화가 끊기지 않도록 설계되었는가?
- 학습자가 주제를 벗어나도 AI가 자연스럽게 원래 상황으로 되돌리는가?

## 16. 최신 반영 상태

이 문서는 다음 최신 의사결정을 반영합니다.

- 홈 시나리오 리스트는 1열 리스트형
- 최소/최대 답변 수는 오늘의 대화 직전 화면과 대화창에서만 표시
- 학습자 답변 수 기준은 레벨별 차등 적용
- 입문만 최대 10회, 초급 이상은 최대 20회
- 콘텐츠 원고는 카테고리별로 검수 후 엑셀에 누적
- 현재 엑셀에는 입문 일상, 여행, 생활 서비스, 건강, 관계 원고가 포함됨
