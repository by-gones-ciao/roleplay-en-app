# 롤플레잉 영어 학습 앱 통합 개발 명세서

- 문서 버전: 1.0
- 기준일: 2026-07-03
- 대상: UX/UI 디자이너, 모바일 앱 개발자, 서버 개발자, AI·콘텐츠 담당자, QA
- 기준 프로토타입: 현재 작업공간의 index.html, styles.css, app.js

---

## 1. 목적

한국인 성인 영어 학습자를 위한 모바일 AI 롤플레잉 앱의 구현 기준을 정의한다. 화면, 콘텐츠, 앱 상태, 온디바이스 AI, 서버 API, 오류 처리, 분석과 QA를 하나의 계약으로 관리한다. 현재 웹 프로토타입은 사용자 흐름 검증용이며 실제 STT·TTS·AI·서버 저장은 제품 개발 단계에서 연결한다.

## 2. 제품 정의

### 대상
- 한국어가 모국어인 성인 입문~중급 학습자
- 영어 지식은 있으나 실제 말하기 부담이 큰 사용자
- 생활·여행 상황을 짧게 연습하려는 사용자

### 핵심 가치
1. 단어와 핵심 표현으로 실전 부담을 낮춘다.
2. 실전과 다른 세부 상황의 미리보기로 표현 전이를 돕는다.
3. AI는 역할을 유지하며 한 번에 질문 하나만 한다.
4. 막히면 직전 AI 발화에 답하는 힌트 두 개를 제공한다.
5. 점수보다 미션 달성과 사용 문장을 중심으로 복습한다.

### 기술 전제
- 모바일 앱 우선
- 온디바이스 Gemma 계열 모델 사용 예정. 정확한 모델·양자화·지원 기기는 검증 후 확정
- STT/TTS는 기기 기능 또는 별도 엔진
- 콘텐츠·추천·학습 기록 동기화는 서버
- 다운로드 콘텐츠와 온디바이스 모델을 활용한 오프라인 학습 우선 검토

## 5. 화면별 명세

### RP-00 롤플레잉 홈
**목적:** 자동 추천 또는 직접 선택으로 학습을 시작한다.

**구성:**
1. 제목 롤플레잉
2. 첫 화면에 노출되는 오늘의 대화 카드
3. 직접 골라보기
4. 카테고리 필터
5. 별도 전체 폭 자유 롤플레잉
6. 난이도 필터
7. 콘텐츠 목록

**카테고리:** 전체, 음식, 쇼핑, 건강, 취미, 여행

**난이도:** 전체, 입문, 초급, 중급

**동작:**
- 오늘의 대화 → 추천 로딩 → RP-10
- 카테고리와 난이도는 AND 조건
- 자유 롤플레잉 → RP-01
- 카드 선택 → 해당 RP-10
- 인기순·최신순은 제공하지 않음

**빈 상태:** 이 조건의 롤플레잉을 준비하고 있어요.

### RP-01 자유 롤플레잉
| 필드 | 필수 | 최대 길이 | 예시 |
|---|---:|---:|---|
| 주제나 상황 | O | 300 | 호텔에서 체크인하고 싶어요. |
| AI 역할 | O | 80 | 호텔 직원 |
| 나의 역할 | O | 80 | 여행객 |

- AI 추천은 초급자용 예시를 채움
- 입력 완료 시 공백·역할 중복·금지 주제를 검사
- 생성 중 중복 제출 차단
- 실패 시 입력값 유지 및 재시도

### RP-10 상황·역할·미션
- 한국어·영어 제목, 난이도, 예상 시간, 일러스트
- 나의 역할 독립 행
- AI의 역할 독립 행
- 미션 1~3개
- 학습 시작하기 → RP-20

### RP-20 유용한 단어
| 입문 | 초급 | 중급 |
|---:|---:|---:|
| 3~4개 | 5개 | 6~7개 |

- 제목 유용한 단어는 한 줄
- 단어, 뜻, 짧은 예문, TTS
- 표현 배우기 → RP-30

### RP-30 유용한 표현
| 입문 | 초급 | 중급 |
|---:|---:|---:|
| 3~4개 | 5개 | 6~7개 |

- 제목 유용한 표현은 한 줄
- 영어, 한국어 뜻, 기본 듣기, 천천히 듣기, 따라 말하기
- CTA는 대화 미리보기
- 건너뛰고 실전 시작 버튼은 없음

### RP-40 대화 미리보기
**목적:** 실전 답을 복사하게 하지 않고 같은 표현을 다른 요청에 적용하는 방법을 보여준다.

- 실전과 장소·역할은 동일
- 세부 요청은 다르게 구성
- 핵심 문장 패턴은 동일
- 4턴 내외, 20~30초
- AI는 한 번에 질문 하나
- 실전 미션의 완성 답을 그대로 노출하지 않음
- 영어 문장 아래 TTS와 번역 아이콘
- 번역 기본 숨김, 아이콘으로 토글
- 상단 건너뛰기 → RP-50
- 하단 CTA 문구는 실전 롤플레잉 → RP-50

### RP-50 실전 준비
- 제목 실전 롤플레잉
- 본문 제목 이제 직접 말해 보세요 한 줄
- 나의 역할·AI의 역할을 각각 독립 행으로 표시
- 미션 목록과 힌트 안내
- CTA 롤플레잉 시작
- 큰 화면에서도 CTA 아래 과도한 빈 공간 금지

### RP-60 실전 롤플레잉
**상단:** 상황 제목, 미션 기반 진행률, 완료 수 0/2, 상황 설명과 미션.

**대화:** 사용자·AI 말풍선, AI 문장 아래 TTS·번역. 번역 기본 숨김.

**하단:** 힌트, 녹음 안내, 종료, 키보드, 음성.

**규칙:**
- 진행률은 턴 수가 아닌 미션 완료 수
- AI가 먼저 시작하고 역할 유지
- 한 번에 질문 하나
- 오프토픽은 원래 상황으로 부드럽게 복귀
- 표현이 달라도 의미가 같으면 미션 인정
- 완료 미션은 이후에도 유지
- 정확도와 미션 판정은 독립

**종료:** 모든 미션 완료, 사용자 종료, 안전 제한, 시스템 오류. 최대 시간·턴은 정책 확정 필요.

### RP-61 힌트
- 사용자가 요청할 때만 바텀시트 표시
- 직전 AI 발화에 바로 답할 수 있는 초급 문장 정확히 2개
- 영어·한국어 번역 제공
- 가능하면 미완료 미션에 도움
- 대화를 진행하거나 미션을 완료하지 않음
- 힌트를 눌러도 자동 전송하지 않음
- 사용자가 직접 말하거나 입력해야 평가

### RP-70 리포트
**상단:** 제목·난이도·미션 결과.

| 지표 | 기준 |
|---|---|
| 말한 단어 | 사용자 영어 단어 수, 반복 포함 |
| 대화 시간 | 실전 시작~종료 |
| 교정 문장 | 문법·문맥 수정이 필요한 문장 |
| 더 좋은 표현 | 맞지만 더 자연스럽게 개선 가능한 문장 |

**상세:** 교정 문장, 더 좋은 표현, 전체 대화, 단어, 표현.

**AI 피드백:** 잘한 점 1개와 개선점 1개. 초급자가 이해할 쉬운 한국어. 과도한 점수화 금지.

## 6. 레벨링 기준

| 기준 | 입문 | 초급 | 중급 |
|---|---|---|---|
| 문장 길이 | 3~6단어 | 5~10단어 | 8~15단어 |
| 단어 | 3~4 | 5 | 6~7 |
| 표현 | 3~4 | 5 | 6~7 |

- 실제 생활·여행 빈도가 높은 표현
- 번역투보다 실제 회화 우선
- 미리보기와 실전 목표 중복 금지

## 7. 앱 상태 모델

~~~text
IDLE → CONTENT_SELECTED → PRE_LEARNING → PREVIEW → READY
→ AI_THINKING → AI_SPEAKING → USER_LISTENING → USER_REVIEW
→ EVALUATING → COMPLETED → REPORT
~~~

별도 상태: HINT_OPEN, TRANSLATION_VISIBLE, KEYBOARD_OPEN, PAUSED, NETWORK_ERROR, MODEL_ERROR, AUDIO_PERMISSION_REQUIRED.

세션 저장값: sessionId, contentId, 단계, 대화 이력, 미션 상태, 힌트 위치, 교정·개선 문장, 시작·누적 시간, 앱·모델·프롬프트 버전.

## 9. AI 계약

### 런타임 입력
~~~json
{
  "requestType": "conversation",
  "hintRequested": false,
  "userRole": "손님",
  "aiRole": "미용사",
  "situation": "미용실에서 대화한다.",
  "missions": ["...", "..."],
  "completedMissions": [],
  "history": [],
  "latestUserMessage": ""
}
~~~

requestType: first_turn, conversation, hint, finalize.

### 응답
~~~json
{
  "en": "",
  "ko": "",
  "correct": "Yes",
  "correct_answer_en": "",
  "correction_explanation": "",
  "mission_complete_0": "No",
  "mission_complete_1": "No",
  "hint_examples_en": [],
  "hint_examples_ko": []
}
~~~

### 필수 규칙
- beginner-level English
- Ask only one question at a time
- 역할·상황 유지
- en에서 교정·번역 금지
- 오프토픽 복귀
- 정확도와 미션 완료 독립 평가
- 한국어 포함 사용자 발화는 correct=No
- 완료된 미션은 No로 되돌리지 않음
- correct=Yes면 교정 필드 빈 문자열
- RFC 8259 JSON만 반환

### 힌트 규칙
- UI 동작으로 처리, 사용자 발화로 평가하지 않음
- 대화 진행 없음
- 직전 AI 발화 답변 두 개
- 영어·한국어 배열 순서 일치
- 미션 상태 유지

### JSON 실패
로컬 복구 → 스키마 검증 → 1회 재요청 → 안전한 기본 대사/재시도 UI. 원문 로그는 개인정보 정책 적용.

## 11. 서버 API 초안

- GET /v1/roleplays?category=&level=&cursor=
- GET /v1/roleplays/{contentId}
- GET /v1/roleplays/daily?level=&timezone=Asia/Seoul
- POST /v1/roleplays/free
- POST /v1/sessions
- PUT /v1/sessions/{sessionId}
- POST /v1/sessions/{sessionId}/complete

공통: 인증, API 버전, 콘텐츠·프롬프트 버전, 페이지네이션, requestId, 오류 코드, 멱등성 키, PII 최소 수집.

## 12. 오프라인·동기화

로컬 저장: 다운로드 콘텐츠, 오늘의 대화 후보, 진행 세션, 미동기화 완료 세션, 모델·프롬프트 버전. 음성 원본은 기본 영구 저장하지 않는 방향을 권장한다.

- 네트워크 복구 시 백그라운드 동기화
- sessionId 멱등 처리
- 진행 세션은 시작 당시 콘텐츠 버전 유지
- 완료 세션 우선 병합

## 13. 오류 처리

| 상황 | 메시지 | 처리 |
|---|---|---|

| 부적절한 자유 입력 | 다른 학습 상황을 입력해 주세요. | 수정 유도 |

## 부록. 미용실 초급 콘텐츠

- 나의 역할: 손님
- AI의 역할: 미용사
- 미션 1: 머리를 조금 다듬고 싶다고 말하기
- 미션 2: 앞머리를 신경 써 달라고 요청하기

단어 5개: trim, hair, bangs, a little, shorter.

표현 5개:
1. I'd like to trim my hair a little.
2. Just a little, please.
3. Please pay special attention to my bangs.
4. Could you make my bangs a little shorter?
5. I'd like to keep my bangs long.

미리보기는 옆머리를 짧게 자르는 요청, 실전은 머리 다듬기와 앞머리 요청이다. 공통 패턴은 I'd like..., A little..., please이다.
