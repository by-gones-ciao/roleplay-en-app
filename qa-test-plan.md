# TalkFlow QA 테스트 계획서

이 문서는 TalkFlow MVP 개발 후 반드시 확인해야 할 테스트 케이스를 정의합니다.

## 1. 테스트 범위

### 포함

- 온보딩
- 홈
- 시나리오 상세
- 오늘의 표현
- 오늘의 단어
- 대화 직전 리마인드
- 실제 대화하기
- 음성/키보드 입력
- 피드백 표시
- 다시 녹음
- 학습 리포트
- 마이 페이지
- 오류/빈 상태

### 제외

- 결제
- 소셜 로그인
- 네이티브 푸시 알림
- 장기 학습 통계

## 2. 테스트 환경

| 환경 | 기준 |
|---|---|
| 모바일 웹 | iPhone Safari, Android Chrome |
| 데스크톱 | Chrome |
| 화면 크기 | 390x844, 375x812, 430x932 |
| 네트워크 | 정상, 느림, 끊김 |
| 권한 | 마이크 허용, 거부 |

## 3. 온보딩 테스트

### TC-ONB-001 레벨 선택

절차:

1. 앱 첫 화면을 연다.
2. Starter~Advanced 중 하나를 선택한다.
3. 다음 화면으로 이동한다.

기대 결과:

- 선택 상태가 명확히 보인다.
- 다음 화면으로 이동한다.
- 선택값이 사용자 설정에 저장된다.

### TC-ONB-002 말하기 속도 선택

절차:

1. 말하기 속도 화면에서 속도를 선택한다.
2. 시작 버튼을 누른다.

기대 결과:

- 선택한 속도가 저장된다.
- 홈 화면으로 이동한다.

## 4. 홈 테스트

### TC-HOME-001 오늘의 대화 시작

절차:

1. 홈에서 오늘의 대화 카드의 바로 시작을 누른다.

기대 결과:

- 시나리오 상세 또는 지정된 시작 화면으로 이동한다.
- 상단 제목과 버튼이 깨지지 않는다.

### TC-HOME-002 마이 페이지 이동

절차:

1. 하단 마이 페이지를 누른다.

기대 결과:

- 마이 페이지로 이동한다.

## 5. 학습 플로우 테스트

### TC-FLOW-001 학습 후 대화하기

절차:

1. 시나리오 상세에서 학습 후 대화하기를 누른다.
2. 오늘의 표현을 확인한다.
3. 오늘의 단어를 확인한다.
4. 오늘의 대화 리마인드를 확인한다.
5. 대화 시작하기를 누른다.

기대 결과:

- 순서가 오늘의 표현 → 오늘의 단어 → 오늘의 대화 → 대화하기로 이어진다.

### TC-FLOW-002 바로 대화하기

절차:

1. 시나리오 상세에서 바로 대화하기를 누른다.

기대 결과:

- 바로 대화하기 화면으로 이동한다.
- 뒤로가기를 누르면 시나리오 상세로 돌아간다.

## 6. 실제 대화 테스트

### TC-LIVE-001 첫 AI 질문 표시

절차:

1. 대화하기 화면에 진입한다.

기대 결과:

- 첫 AI 질문이 표시된다.
- AI 음성 버튼과 번역 버튼이 보인다.
- 대화 진행은 0/10으로 표시된다.

### TC-LIVE-002 키보드 입력

절차:

1. 키보드 버튼을 누른다.
2. 영어 문장을 입력한다.
3. 보내기를 누른다.

기대 결과:

- 입력창이 대화 내용을 완전히 가리지 않는다.
- 학습자 말풍선이 표시된다.
- 다음 AI 질문이 표시된다.

### TC-LIVE-003 빈 입력 방지

절차:

1. 키보드 입력창을 연다.
2. 아무것도 입력하지 않고 보내기를 누른다.

기대 결과:

- `영어 문장을 입력해 주세요.` 문구가 표시된다.
- 빈 말풍선이 생성되지 않는다.

### TC-LIVE-004 perfect 피드백

절차:

1. 완벽한 답변을 입력한다.

기대 결과:

- 초록 체크 아이콘이 표시된다.
- 아이콘을 눌러도 설명 팝업이 열리지 않는다.

### TC-LIVE-005 better 피드백

절차:

1. better로 평가될 답변을 입력한다.
2. 손바닥 아이콘을 누른다.

기대 결과:

- 한글 설명, 구분선, 더 좋은 표현, 영어 문장 순서로 표시된다.

### TC-LIVE-006 correction 피드백

절차:

1. correction으로 평가될 답변을 입력한다.
2. 느낌표 아이콘을 누른다.

기대 결과:

- 한글 설명, 구분선, 교정 문장, 영어 문장 순서로 표시된다.

### TC-LIVE-007 다시 녹음

절차:

1. 3턴 이상 대화한다.
2. 2번째 학습자 말풍선의 다시 녹음 버튼을 누른다.

기대 결과:

- 2번째 학습자 답변 이후 대화가 삭제된다.
- 같은 AI 질문에 다시 답할 수 있다.
- 미션/리포트 데이터가 재계산된다.

## 7. 턴 규칙 테스트

### TC-TURN-001 0턴 종료

절차:

1. 대화하기에 진입한다.
2. 아무 말도 하지 않고 X를 누른다.
3. 나가기를 누른다.

기대 결과:

- 리포트가 생성되지 않는다.
- 직전 진입 화면으로 돌아간다.

### TC-TURN-002 1~9턴 종료

절차:

1. 1~9턴만 대화한다.
2. X를 누른다.
3. 리포트 보기를 누른다.

기대 결과:

- 짧은 리포트가 생성된다.
- 미션 평가는 실제 완료 기준으로 표시된다.

### TC-TURN-003 미션 완료 + 10턴 이상

절차:

1. 미션을 모두 완료한다.
2. 10턴 이상 대화한다.

기대 결과:

- 마무리/더 대화 선택지가 표시된다.

### TC-TURN-004 더 대화하기

절차:

1. 더 대화하기를 누른다.

기대 결과:

- 20턴까지 대화가 확장된다.
- 진행 표시는 20턴 기준으로 바뀐다.

### TC-TURN-005 18턴 마무리 예고

절차:

1. 18턴에 도달한다.

기대 결과:

- AI가 마무리 예고를 한다.

### TC-TURN-006 20턴 자동 종료

절차:

1. 20턴에 도달한다.

기대 결과:

- AI가 최종 종료 문구를 표시한다.
- 학습 리포트로 이동한다.

## 8. 리포트 테스트

### TC-REPORT-001 완료 리포트

절차:

1. 미션을 모두 완료하고 리포트로 이동한다.

기대 결과:

- reportType이 complete이다.
- 미션 평가는 2/2로 표시된다.

### TC-REPORT-002 교정 문장 보기

절차:

1. correction이 있는 리포트에서 교정 문장 보기를 누른다.

기대 결과:

- 실제 대화창 형식으로 표시된다.
- 다시하기 버튼은 없다.

### TC-REPORT-003 더 좋은 표현 보기

절차:

1. better가 있는 리포트에서 더 좋은 표현 보기를 누른다.

기대 결과:

- 실제 대화창 형식으로 표시된다.
- 다시하기 버튼은 없다.

### TC-REPORT-004 전체 대화 보기

절차:

1. 전체 대화 보기를 누른다.

기대 결과:

- 전체 대화가 표시된다.
- AI 발화에는 음성/번역 버튼이 있다.
- 학습자 평가 아이콘과 다시하기 버튼은 없다.

## 9. 마이 페이지 테스트

### TC-PROFILE-001 레벨 변경

절차:

1. 마이 페이지에서 레벨 선택을 누른다.
2. 다른 레벨을 선택하고 확인한다.

기대 결과:

- 레벨이 저장된다.

### TC-PROFILE-002 말하기 속도 변경

절차:

1. 말하기 속도를 누른다.
2. 다른 속도를 선택하고 확인한다.

기대 결과:

- 속도가 저장된다.
- AI 음성 재생 속도에 반영된다.

### TC-PROFILE-003 알림 시간 변경

절차:

1. 알림을 누른다.
2. 오전/오후, 시간, 분을 선택한다.
3. 확인한다.

기대 결과:

- 선택 시간이 저장된다.

## 10. 오류 테스트

### TC-ERR-001 마이크 권한 거부

기대 결과:

- 권한 안내 문구가 표시된다.
- 키보드 입력으로 대체할 수 있다.

### TC-ERR-002 음성 인식 실패

기대 결과:

- 다시 말하기와 키보드 입력이 제공된다.

### TC-ERR-003 AI 응답 실패

기대 결과:

- 다시 시도 버튼이 제공된다.
- 1턴 이상이면 리포트 보기 선택지가 제공된다.

### TC-ERR-004 리포트 생성 실패

기대 결과:

- 다시 시도와 홈으로 가기 버튼이 제공된다.

## 11. 시각 QA

1. 모든 버튼 텍스트가 잘리지 않는가?
2. 모바일 화면에서 요소가 겹치지 않는가?
3. 키보드 입력창이 대화창을 과도하게 가리지 않는가?
4. 대화 진행 표시가 너무 복잡하지 않은가?
5. 피드백 아이콘 크기가 일관적인가?
6. 홈/뒤로가기/X 버튼의 의미가 혼동되지 않는가?

## 12. 출시 전 통과 기준

- 필수 테스트 케이스 90% 이상 통과
- 0턴/1~9턴/10턴/20턴 종료 케이스 모두 통과
- 마이크 실패 시 키보드 대체 가능
- 리포트 수치 계산 오류 없음
- 모바일 주요 화면에서 텍스트 잘림 없음
