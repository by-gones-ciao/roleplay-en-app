# TalkFlow UI Design Reference

이 문서는 TalkFlow 화면을 개발할 때 참고할 디자인 원칙만 정리한 문서입니다. 외부 앱 스크린샷 원본 이미지는 저작권 및 공개 저장소 노출 이슈를 피하기 위해 GitHub에 포함하지 않습니다. 개발자는 아래 원칙을 TalkFlow의 제품 구조와 톤에 맞게 재해석해야 하며, 특정 앱 화면을 그대로 복제하지 않습니다.

## 1. 전체 톤

- 성인 영어 학습자가 부담 없이 말하기를 시작할 수 있는 밝고 자신감 있는 분위기를 유지한다.
- 학습 앱이지만 과하게 어린 느낌은 피하고, 단순하고 선명한 모바일 서비스 느낌을 우선한다.
- 화면마다 핵심 행동이 하나씩 분명하게 보이도록 구성한다.
- 설명 텍스트는 짧게 유지하고, 가능한 경우 숫자, 상태, 버튼, 아이콘으로 의미를 전달한다.

## 2. 색상 방향

- 주요 브랜드 컬러는 선명한 그린 계열을 중심으로 사용한다.
- 보조 배경은 연한 민트, 아이보리, 아주 밝은 회색을 사용해 화면 피로도를 낮춘다.
- CTA 버튼은 그린 또는 노랑 계열처럼 즉시 눈에 들어오는 색을 사용한다.
- 오류, 교정, 주의는 붉은색을 쓰되 전체 면을 과하게 채우기보다 아이콘, 테두리, 작은 강조에 우선 적용한다.
- 정답, 완료, 성공은 그린을 사용한다.
- 비활성 상태는 밝은 회색과 낮은 대비로 표현한다.

## 3. 서체와 글자 위계

- 한글과 영문 모두 둥글고 현대적인 산세리프 계열을 사용한다.
- 제목은 굵고 크게, 본문은 가볍고 읽기 쉬운 크기로 구분한다.
- 한 화면 안에서 글자 크기 단계는 3~4개 이내로 제한한다.
- 모바일 화면에서는 긴 문장을 줄이고, 버튼 안 텍스트가 절대 잘리지 않도록 한다.
- 학습 콘텐츠의 핵심 표현은 볼드와 컬러로 강조한다.
- 오늘의 표현은 핵심 패턴/표현에 색상과 볼드를 적용한다.
- 오늘의 단어는 학습 단어에 색상과 볼드를 적용한다.

## 4. 화면 구성 원칙

- 모바일 앱 기준의 세로 화면을 우선한다.
- 상단에는 현재 위치와 나가기/뒤로가기 같은 핵심 내비게이션만 둔다.
- 홈의 시나리오 목록은 1열 리스트형으로 구성해 제목과 상황을 충분히 읽을 수 있게 한다.
- 카드형 UI는 정보 묶음, 선택지, 리포트 요약처럼 실제로 묶어 보여야 할 때만 사용한다.
- 카드 안에 또 다른 카드를 중첩하지 않는다.
- 여백은 충분히 주되, 핵심 버튼이 지나치게 아래로 밀리지 않게 한다.
- 대화 직전 화면은 정보 카드보다 가벼운 리마인드처럼 보여야 한다.

## 5. 대화 화면 원칙

- 실제 대화하기 화면에서는 학습자가 지금 무엇을 해야 하는지 즉시 알 수 있어야 한다.
- 대화 진행 상태는 텍스트보다 숫자 중심으로 보여준다.
- 학습자 답변 수 기준을 화면에 반영한다.
  - 입문: 최소 5번, 최대 10번
  - 초급: 최소 5번, 최대 20번
  - 초중급: 최소 6번, 최대 20번
  - 중급: 최소 7번, 최대 20번
  - 고급: 최소 8번, 최대 20번
- 진행바는 과도하게 세분화하지 않고, 오른쪽 끝에 `8/10`처럼 현재 상태를 명확히 표시한다.
- 미션 완료 여부는 번호가 체크 상태로 바뀌는 방식처럼 직관적으로 표현한다.
- 중간 종료는 부담 없이 선택할 수 있게 하되, 리포트 확인 흐름을 자연스럽게 제공한다.

## 6. 피드백 UI 원칙

- 학습자 발화 평가는 완벽, 더 좋은 표현, 교정 문장으로 구분한다.
- 완벽한 문장에는 설명 팝업을 제공하지 않아도 된다.
- 더 좋은 표현과 교정 문장은 대화 버블 아래에 한눈에 볼 수 있는 피드백 박스로 보여준다.
- 피드백 박스의 구조는 설명 → 구분선 → 제목 → 제안 문장 순서를 기본으로 한다.
- 설명은 한국어로 짧고 친절하게 제공한다.
- 제목은 구분 색상과 볼드로 강조한다.
- 피드백 아이콘은 작고 균일해야 하며, 대화 버블보다 시선을 빼앗지 않아야 한다.

## 7. 버튼과 인터랙션

- 주요 버튼은 하단에 크게 배치하되, 한 화면 안에서 가장 중요한 행동 하나가 먼저 보이게 한다.
- 보조 행동은 테두리 버튼이나 옅은 배경 버튼으로 구분한다.
- 마이크, 재녹음, 듣기, 번역, 리포트 보기처럼 반복적으로 쓰이는 기능은 아이콘 기반으로 설계한다.
- 아이콘은 같은 굵기와 같은 시각 톤을 유지한다.
- 재녹음 버튼은 키보드 입력 화면을 여는 기능이 아니라 학습자 답변을 지우고 다시 말할 수 있는 기능이어야 한다.

## 8. 온보딩과 마이 페이지

- 온보딩은 영어 말하기 수준 선택과 말하기 속도 선택을 별도 화면으로 나눈다.
- 선택지는 길게 설명하되, 화면에서 잘리지 않도록 줄 간격과 버튼 높이를 충분히 확보한다.
- 마이 페이지에서는 레벨 선택, 말하기 속도, 알림 시간을 일관된 방식으로 수정할 수 있어야 한다.
- 알림 시간은 실제 알람 시계처럼 오전/오후, 시간, 분을 휠 피커 형태로 선택하는 방향을 우선한다.
- 설정 변경 화면에는 확인 버튼 사용 여부를 통일한다.

## 9. 리포트 화면 원칙

- 학습 리포트는 학습자가 짧게 대화하고 중단해도 항상 제공한다.
- 진행 정도에 따라 리포트의 톤과 내용 밀도를 조절한다.
- 교정 문장 보기, 더 좋은 표현 보기, 전체 대화 보기는 실제 대화창 형식으로 보여준다.
- 전체 대화 보기에서는 AI 발화의 번역과 음성 기능도 대화창과 동일하게 반영한다.
- 리포트 요약 수치는 단어 수, 대화 시간, 미션 평가가 같은 시각 톤으로 보이게 한다.

## 10. 개발 적용 시 주의사항

- 참고 이미지는 분위기, 비율, 위계, 여백, 색감, 컴포넌트 감각을 이해하기 위한 자료로만 사용한다.
- 외부 앱의 캐릭터, 일러스트, 고유 아이콘, 화면 구성을 그대로 복제하지 않는다.
- TalkFlow의 핵심 경험은 AI 롤플레잉, 미션 완료, 발화 피드백, 학습 리포트이므로 모든 UI는 이 흐름을 돕는 방향으로 판단한다.
- 개발 중 새 화면이 추가되면 이 문서의 원칙과 충돌하지 않는지 확인한다.
