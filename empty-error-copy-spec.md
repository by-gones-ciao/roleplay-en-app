# TalkFlow 빈 상태/오류/권한 문구 기준서

이 문서는 TalkFlow에서 사용자가 막히거나, 데이터가 없거나, 권한/네트워크/API 문제가 발생했을 때 보여줄 문구와 버튼 기준을 정의합니다.

## 1. 공통 문구 원칙

1. 사용자를 탓하지 않는다.
2. 원인을 짧게 말한다.
3. 다음 행동을 바로 제시한다.
4. 한국어 문구는 친절하고 차분한 톤으로 작성한다.
5. 가능한 한 한 문장으로 끝낸다.
6. 버튼은 1~2개를 기본으로 한다.
7. 기술적인 오류명은 사용자에게 직접 노출하지 않는다.

## 2. 문구 구조

기본 구조:

```text
상황을 짧게 설명해요. 다음 행동을 안내해요.
```

예:

```text
말소리가 잘 들리지 않았어요. 다시 말해 주세요.
```

버튼:

- 다시 시도
- 키보드로 입력

## 3. 마이크 권한

### 권한 요청 중

문구:

```text
마이크 권한을 확인하고 있어요.
```

버튼:

- 없음

### 권한 거부

문구:

```text
마이크 권한이 꺼져 있어요. 권한을 허용하거나 키보드로 입력해 주세요.
```

버튼:

- 다시 시도
- 키보드로 입력

### 권한을 다시 요청할 수 없는 경우

문구:

```text
브라우저 설정에서 마이크 권한을 허용해 주세요.
```

버튼:

- 키보드로 입력

## 4. 음성 인식 실패

| 상황 | 문구 | 버튼 |
|---|---|---|
| 아무 말도 감지되지 않음 | 말소리가 잘 들리지 않았어요. 다시 말해 주세요. | 다시 말하기 / 키보드로 입력 |
| 너무 짧음 | 조금만 더 길게 말해 주세요. | 다시 말하기 / 키보드로 입력 |
| 인식 불명확 | 정확히 듣지 못했어요. 다시 말하거나 키보드로 입력해 주세요. | 다시 말하기 / 키보드로 입력 |
| 영어가 거의 없음 | 영어로 한 문장 말해 주세요. | 다시 말하기 / 키보드로 입력 |
| 소음이 큼 | 주변 소리가 커서 잘 듣지 못했어요. 다시 말해 주세요. | 다시 말하기 / 키보드로 입력 |

## 5. 키보드 입력

### 빈 입력

문구:

```text
영어 문장을 입력해 주세요.
```

버튼:

- 없음

### 너무 긴 입력

문구:

```text

```

버튼:

- 없음

### 영어가 아닌 입력

문구:

```text
영어로 입력해 주세요.
```

버튼:

- 없음

## 6. AI 응답 생성 실패

### 다음 질문 생성 실패

문구:

```text
다음 질문을 가져오지 못했어요. 다시 시도해 주세요.
```

버튼:

- 다시 시도
- 리포트 보기

조건:

- `리포트 보기`는 완료 턴이 1턴 이상 있을 때만 제공한다.

### AI 응답이 오래 걸리는 경우

1초 이후:

```text
다음 질문을 준비하고 있어요.
```

8초 이후:

```text
응답이 조금 늦어지고 있어요. 다시 시도할 수 있어요.
```

버튼:

- 다시 시도

## 7. 학습자 발화 평가 실패

문구:

```text
표현을 확인하지 못했어요. 다시 시도해 주세요.
```

버튼:

- 다시 시도
- 키보드로 입력

주의:

- 평가 실패 시 학습자 발화를 바로 삭제하지 않는다.
- 재시도할 수 있게 유지한다.

## 8. 리포트 생성 실패

문구:

```text
학습 리포트를 만들지 못했어요. 다시 시도해 주세요.
```

버튼:

- 다시 시도
- 홈으로 가기

조건:

- 대화 기록이 저장되어 있으면 다시 시도 가능해야 한다.

## 11. 네트워크 오류

### 공통 문구

```text
연결이 불안정해요. 잠시 후 다시 시도해 주세요.
```

### 상황별 버튼

| 위치 | 버튼 |
|---|---|
| 대화 중 | 다시 시도 / 리포트 보기 |
| 음성 인식 중 | 다시 말하기 / 키보드로 입력 |
| 리포트 생성 중 | 다시 시도 / 홈으로 가기 |
| 시나리오 생성 중 | 다시 시도 |

### 설정 저장 실패


버튼:

- 다시 시도

## 13. 나만의 시나리오 입력 오류

### 상황 설명 없음

```text
상황이나 주제 설명을 입력해 주세요.
```

### 나의 역할 없음

```text
나의 역할을 입력해 주세요.
```

### AI 역할 없음

```text
AI의 역할을 입력해 주세요.
```

### 전체 입력 부족

```text
상황과 역할을 모두 입력해 주세요.
```

### 생성 실패

```text
대화 시나리오를 만들지 못했어요. 다시 시도해 주세요.
```

버튼:

- 다시 시도

## 14. 빈 상태

### 교정 문장 없음

```text
이번 대화에서는 교정이 필요한 문장이 없었어요.
```

### 더 좋은 표현 없음

```text
이번 대화에서는 더 좋은 표현 제안이 없었어요.
```

### 학습 기록 없음

```text
아직 학습 기록이 없어요.
```

### 알림 시간 미설정

```text
알림 시간이 아직 설정되지 않았어요.
```

### 검색 결과 없음

```text
조건에 맞는 대화가 없어요.
```

## 15. 대화 종료 확인 문구

### 0턴 종료

문구:

```text
아직 대화를 시작하지 않았어요. 나가시겠어요?
```

버튼:

- 계속 대화하기
- 나가기

### 1턴 이상 종료

문구:

```text
대화를 여기까지 할까요?
```

버튼:

- 계속하기
- 리포트 보기

### 미션 완료 + 5턴 이상

문구:

```text
미션을 모두 완료했어요!
마무리할까요 아니면 대화를 좀 더 할까요?
```

버튼:

- 마무리하고 리포트 보기
- 조금 더 대화하기

## 17. 토스트 문구

짧은 성공/상태 알림은 토스트로 표시한다.

| 상황 | 문구 |
|---|---|
| AI 추천 입력 완료 | AI 추천 내용이 입력되었어요. |
| 맞춤 시나리오 생성 완료 | 입력한 내용으로 학습을 만들었어요. |

| 속도 저장 완료 | 말하기 속도가 변경되었어요. |
| 알림 저장 완료 | 알림 시간이 변경되었어요. |
| 준비 중 기능 | 이 기능은 곧 추가될 예정이에요. |

## 18. 문구 톤 가이드

### 좋은 문구

```text
말소리가 잘 들리지 않았어요. 다시 말해 주세요.
```

이유:

- 사용자 탓을 하지 않음
- 문제를 짧게 설명
- 다음 행동을 안내

### 피해야 할 문구

```text
오류가 발생했습니다. Error code: STT_400
```

이유:

- 기술적임
- 사용자가 무엇을 해야 하는지 알 수 없음

## 20. 오류 코드 제안

| 코드 | 의미 |
|---|---|
| mic_permission_denied | 마이크 권한 거부 |
| speech_no_input | 음성 없음 |
| speech_too_short | 발화가 너무 짧음 |
| speech_unclear | 음성 인식 불명확 |
| speech_language_mismatch | 영어 입력 부족 |
| ai_response_failed | AI 응답 생성 실패 |
| evaluation_failed | 발화 평가 실패 |
| report_failed | 리포트 생성 실패 |
| tts_failed | 음성 재생 실패 |
| translation_failed | 번역 실패 |
| network_unstable | 네트워크 불안정 |
| save_failed | 저장 실패 |
| scenario_generation_failed | 시나리오 생성 실패 |

## 21. 품질 검수 체크리스트

1. 모든 오류 문구가 사용자를 탓하지 않는가?
2. 모든 오류 문구에 다음 행동이 있는가?
3. 마이크 권한 거부 시 키보드 입력으로 우회할 수 있는가?
4. 음성 인식 실패 시 다시 말하기가 가능한가?
5. AI 응답 실패 시 대화가 완전히 막히지 않는가?
6. 1턴 이상 대화 후 오류가 나면 리포트 보기 선택지가 있는가?
7. 0턴 종료 시 리포트를 제공하지 않는가?
8. 빈 상태 문구가 무섭거나 차갑지 않은가?
9. 기술 오류 코드가 사용자에게 직접 노출되지 않는가?
10. 버튼 문구가 짧고 명확한가?
